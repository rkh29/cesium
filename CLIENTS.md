# 卫星异常检测系统 - 客户端说明

## 系统架构

本系统包含 **3 个前端应用**，共享同一套 API 和 Store：

1. **Web 管理端** (`/`) - 完整功能的管理界面
2. **运维操作端** (`/ops/*`) - 侧重操作和控制功能
3. **监控展示端** (`/monitor/*`) - 只读监控功能

---

## 访问方式

### Web 管理端
- **URL**: `http://localhost:3000/`
- **功能**: 完整的管理功能，包括监控、操作、配置等
- **用户角色**: 系统管理员

### 运维操作端
- **URL**: `http://localhost:3000/ops`
- **功能**: 
  - 故障注入
  - 卫星操作控制
  - 工作流执行
  - AI 助手
  - 安全审计
- **用户角色**: 运维操作员

### 监控展示端
- **URL**: `http://localhost:3000/monitor`
- **功能**:
  - 实时状态监控
  - 资源指标查看
  - 异常告警查看
  - 异常列表
  - 趋势分析
- **用户角色**: 监控观察员（只读）

---

## 技术实现

### 路由配置
- 所有路由定义在 `src/router/index.ts`
- 客户端路由使用嵌套路由，每个客户端有自己的布局组件

### 布局组件
- **Web 管理端**: 使用 `App.vue` 中的布局（带 Cesium 3D 视图）
- **运维操作端**: `src/layouts/OpsLayout.vue`
- **监控展示端**: `src/layouts/MonitorLayout.vue`

### 共享资源
- **API**: `src/api/index.ts` - 所有客户端共享同一套 API
- **Store**: `src/stores/*` - 所有客户端共享同一套状态管理
- **组件**: `src/components/*` - 所有客户端可以复用组件

---

## 开发说明

### 添加新页面到运维操作端
1. 在 `src/views/ops/` 下创建页面组件
2. 在 `src/router/index.ts` 的 `/ops` 路由下添加子路由

### 添加新页面到监控展示端
1. 在 `src/views/monitor/` 下创建页面组件
2. 在 `src/router/index.ts` 的 `/monitor` 路由下添加子路由

### 样式规范
- 所有客户端使用相同的 CSS 变量（定义在 `App.vue` 的 `<style>` 中）
- 运维操作端主色调：橙色/黄色（`#FFD04B`）
- 监控展示端主色调：绿色（`#00FF88`）
- Web 管理端主色调：蓝色（`#00D4FF`）

## 部署说明

所有三个客户端都在同一个 Vite 应用中，通过路由区分。部署时：
- 生产环境：构建一次，三个客户端都可以访问
- 开发环境：`npm run dev` 启动后，访问不同路径即可

---

## 后续扩展

如果需要将客户端拆分为独立应用：
1. 创建独立的 `vite.config.ops.ts` 和 `vite.config.monitor.ts`
2. 创建独立的入口文件 `main-ops.ts` 和 `main-monitor.ts`
3. 配置多入口构建

但目前通过路由区分的方式更简单，维护成本更低。
